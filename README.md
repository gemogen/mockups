# mockups — общая веб-галерея дизайн-макетов

Живая галерея: **https://gemogen.github.io/mockups/**

Постоянная ссылка в интернете на все макеты экранов всех приложений. Дизайн всегда
показывается отсюда, а не из локальных HTML-файлов. Открывается с телефона.

## Как устроено

- `index.html` — сама галерея (data-driven, тёмная тема). Читает `manifest.json`.
- `manifest.json` — список приложений и экранов (единственный источник правды).
- `<slug>/<screen>.html` — self-contained макет экрана (всё inline/base64).
- `<slug>/sketch/<screen>.png` — опциональный исходный эскиз (для режима сравнения).

Добавить макет = положить HTML + дописать строку в `manifest.json`. `index.html` править не нужно.

## Как добавить макет

Из корня клона этого репозитория:

```powershell
./publish-mockup.ps1 -App garage -Name "Гараж" -Icon "🚗" `
  -Screen to -Title "ТО — таймлайн" `
  -Html ".\to.html" -Sketch ".\sketch_to.png" -Status "на согласовании"
```

Скрипт сам: скопирует HTML (и эскиз) в нужную папку, обновит `manifest.json`,
сделает commit + push. Через ~1 минуту (CDN) макет виден в галерее — нажми «↻ Обновить».

Параметры: `-App` slug, `-Screen` id, `-Title` название, `-Html` путь к макету
(обязательные); `-Name`/`-Icon` для новой карточки приложения, `-Sketch` эскиз,
`-Status` бейдж, `-NoPush` собрать без отправки (необязательные).

## Из облачной сессии (claude.ai/code)

Чтобы пушить макеты из облака, репозиторий `gemogen/mockups` должен быть в списке
репозиториев той облачной сессии (иначе отправку запретят). Добавляется один раз.

## Схема manifest.json

```json
{
  "apps": [
    {
      "slug": "garage", "name": "Гараж", "icon": "🚗",
      "screens": [
        { "id": "to", "title": "ТО — таймлайн", "file": "garage/to.html",
          "sketch": "garage/sketch/to.png", "status": "на согласовании",
          "updated": "2026-06-22" }
      ]
    }
  ]
}
```

`sketch` — необязательное поле. Если эскиза нет — в галерее показывается только макет.
